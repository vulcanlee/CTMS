# 找出儀錶板中，每個頁籤的資料關係，準備用於資料匯出成為一個大表之用

在 CTMS 這個專案內，將會用來記錄婦產科癌症臨床試驗的各種醫療紀錄，這裡採用的是儀表板的觀念來設計，也就是把病人的每個不同資訊，記錄在不同的頁籤來呈現，當開啟一個病人資料後，將會透過 BasicClinicalPage2.razor 頁面 (這裡的頁籤只有一個，稱作 臨床資訊)來呈現出這個病人的基本資訊、癌症類型與癌症期別等資訊，在這個頁面最下方，將會有 BottomNavigationView.razor 這個元件，這裡可以切換到 臨床資料 , 抽血資料 , 副作用 , 問卷 , 追蹤資料 與 風險評估 等不同的頁面，這些頁面內又會擁有不同的頁籤，如在 臨床資料 頁面內，將會有 手術 、 病理報告 、 化學治療 、 合併用藥 與 病史記錄 等不同的頁籤

由於現在需要把所有參與計劃的病人和不同種的醫療紀錄，需要匯出成為一個大 csv 表格，這裡需要先把每個頁籤內的資料關係找出來，才能夠把資料匯出成為一個大表格，這個大表格的 csv 內容，將會提交給統計專案來進行分析與研究。

除了 臨床資訊 & 風險評估 這兩個頁面內僅會有一筆紀錄且沒有其他頁籤存在，在其他頁面的各種頁籤內，都會有個 Visit Code 欄位，這個欄位是用來區分同一個病人在不同時間點的紀錄，每個 Visit Code 都會有一個該頁籤的所有欄位紀錄。

Visit Code 對應到的頁籤內的資料，又具備有兩種關聯，第一種，將會是一個 Visit Code 僅會對應到一個頁籤內的所有欄位，也就是僅有一筆紀錄，例如：臨床資料 頁面內，對於這些頁籤: 手術 、 病理報告 、 病史記錄 這三個頁籤，每個 Visit Code與這些頁籤內的紀錄，呈現的是一對一的關係，而對於 化學治療 、 合併用藥 這兩個頁籤，每個 Visit Code 與這些頁籤內的紀錄，呈現的是一對多的關係，也就是說，每個 Visit Code 可能會對應到多筆紀錄，例如：同一個病人在同一個 Visit Code 可能會有多筆化學治療紀錄，或是多筆合併用藥紀錄。

對於 於 化學治療 、 合併用藥 這兩個頁籤的紀錄，其紀錄數量大小，將會不確定，因為這要看當如個管師當時輸入了幾筆紀錄。

對於抽血頁面，因為這個系統將會記錄三家醫院的癌症病人資訊，每家醫院對於抽血檢驗項目的數量也不相同，此時若要匯出成為大表時，大表內的每個 Column 都具有同樣欄位，面對數量不同，項目不同，如何產生一致性的大表，這個問題需要考量。在問卷頁面內，將會有 化療副作用 、 標靶副作用 、 放療副作用 、 WHOQOL 問卷 、 個人史 、 家族史 、 生活品質 與 健康 等八個問卷頁籤，每個問卷頁籤內的題目與子題目數量大小。另外，每次的 Visit Code，不見得都要填寫所有的問卷，也許只需要填寫幾個問卷，這樣，在進行匯出成為 csv 格式時候，對於沒有填寫的問卷，需要保留空白。

由於需要將這些資料全部都匯出成為一個大表格，因此需要先把這些資料的關係找出來，才能夠把資料匯出成為一個大表格，而在設計上，想要採用分批處理方式來設計程式碼，也就是將 臨床資訊、臨床資料、抽血資料、副作用、問卷、追蹤資料與風險評估 等七個頁面內的資料，分別產生出已經準備要匯出的 csv 資料，接著再把這些資料合併成為一個大表格

所有的臨床治療紀錄，都已非結構的 JSON 方式做儲存，在 C# 內，將會採用 PatientData 這個結構來記錄，臨床資訊 定義在這個類別內，其他頁面、頁籤的資料與欄位，可以從 Main臨床資料 這個類別內找到
