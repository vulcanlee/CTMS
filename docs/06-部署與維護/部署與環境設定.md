# 部署與環境設定

說明 CTMS 主站與 AIAgent 背景服務的執行環境、設定檔與重要參數。

## 1. 環境需求

- 作業系統：**Windows**（專案使用 Windows 專用套件與 `C:\...` 路徑）。
- .NET SDK 9.x。
- Visual Studio 2022 以上，或 `dotnet CLI`。
- 可寫入本機檔案系統的權限（上傳、佇列、推論輸出目錄）。

為何偏 Windows：`Syncfusion.HtmlToPdfConverter.Net.Windows`（PDF 轉換）、預設 `C:\...` 路徑、DICOM/檔案處理流程皆以 Windows 為前提。部署 Linux 前需檢查 PDF 轉換、檔案路徑與影像處理依賴。

## 2. 快速啟動

```powershell
dotnet restore .\src\CTMS\CTMS.sln
dotnet build   .\src\CTMS\CTMS.sln

# 主系統（Web）
dotnet run --project .\src\CTMS\CTMS\CTMS.csproj

# AI 背景服務
dotnet run --project .\src\CTMS\AIAgent\AIAgent.csproj
```

初次啟動會自動建立/更新 SQLite schema、建立預設專案與角色、建立預設管理者帳號 `support`（密碼同為 `support`）。

> 安全提醒：預設帳密僅適合開發。正式環境須立即變更，並將敏感資訊（Syncfusion 授權、AI endpoint/API key）移出原始碼與 `appsettings.json`。

## 3. 設定檔位置

| 檔案 | 用途 |
| --- | --- |
| [src/CTMS/CTMS/appsettings.json](../../src/CTMS/CTMS/appsettings.json) | 主站設定 |
| `src/CTMS/CTMS/appsettings.Development.json` | 主站開發環境覆寫 |
| [src/CTMS/AIAgent/appsettings.json](../../src/CTMS/AIAgent/appsettings.json) | AIAgent 設定 |
| `src/CTMS/AIAgent/appsettings.Development.json` | AIAgent 開發環境覆寫 |

## 4. 主站設定區塊（CTMSSettings）

- `SyncfusionLicenseKey`：Syncfusion 授權金鑰。
- `ConnectionStrings.DefaultConnection`：SQL Server LocalDB 連線（替代）。
- `ConnectionStrings.SQLiteDefaultConnection`：`Data Source=BackendDB.db`（預設使用）。
- `SystemInformation.SystemName` / `SystemDescription`：系統名稱（「AI 臨床試驗管理平臺」）。
- `SystemInformation.SystemVersion`：**版本號**。

### 版本號規則（重要）

依專案規範（[CLAUDE.md](../../CLAUDE.md)）：**每產生一次建置內容，`SystemVersion` 的最後一碼（Patch）加 1**（例：`1.1.216 → 1.1.217`，不進位、不分異動性質），並於 commit message 說明版本變更。

## 5. AgentSetting（AI 佇列與路徑）

主站與 AIAgent 各有一份 `AgentSetting`，定義 DICOM/AI 推論管線的資料夾與佇列。主站版本主要差異是風險評估模型路徑，AIAgent 版另含卵巢癌（OC）風險模型路徑。

| 設定鍵 | 範例值 | 用途 |
| --- | --- | --- |
| `DicomFolderPath` | `C:\temp\Dicom` | DICOM 來源資料夾 |
| `QueueFolderPath` | `C:\temp\CTMS\Queue` | 佇列根目錄 |
| `InBoundQueueName` | `InBound` | 進件佇列 |
| `Phase1QueueName` / `Phase1WaitingQueueName` | `Phase1` / `Phase1Waiting` | 第一階段佇列 |
| `Phase2QueueName` / `Phase2WaitingQueueName` | `Phase2` / `Phase2Waiting` | 第二階段佇列 |
| `Phase3QueueName` / `Phase3WaitingQueueName` | `Phase3` / `Phase3Waiting` | 第三階段佇列 |
| `OutBoundQueueName` / `CompleteQueueName` | `OutBound` / `Complete` | 輸出與完成佇列 |
| `Phase1TmpFolder` / `Phase2TmpFolder` | `C:\CanWellBeing\temp1` / `temp2` | 各階段暫存 |
| `InferencePath` | `C:\CanWellBeing\inference` | AI 推論程式路徑 |
| `風險評估模型` | `C:\EndometrioidCancer` | 內膜癌風險模型目錄 |
| `風險評估模型OC` | （AIAgent 設定） | 卵巢癌風險模型目錄 |

詳細管線行為見 [../05-AI推論/AIAgent背景服務.md](../05-AI推論/AIAgent背景服務.md) 與 [../05-AI推論/overview.md](../05-AI推論/overview.md)。
