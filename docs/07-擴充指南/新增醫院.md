# 擴充指南：新增醫院

本指南整理 CTMS 系統中所有與「醫院」綁定的程式與資料定義點，提供新增一家或多家醫院時的完整修改清單與操作順序。內容全部來自實際原始碼盤點。

## 1. 背景

系統目前支援三家合作醫院：

| 代碼（Prefix） | Sheet 短名 | 院別全名 |
| --- | --- | --- |
| `NCKUH` | 成大 | 成大醫院（國立成功大學醫學院附設醫院） |
| `CHIMEIH` | 奇美 | 奇美醫院 |
| `KGH` | 郭綜合 | 郭綜合醫院 |

依需求未來將再新增三家醫院。由於醫院代碼、院別短名、受試者編號前綴、隨機表分頁、抽血檢驗檔名、儀表板分組等多處以「三家硬編碼」實作（多為 `switch` / 連續 `if` / 固定清單），新增醫院屬於橫跨多檔的變更，務必依本清單逐點處理。

備註：

- **權限 / 專案關聯**：盤點未發現以醫院為單位的角色權限或專案關聯設定，新增醫院不需異動權限模型（若日後導入須另行評估）。
- **下拉選單只有一處**：新增病人對話框 [AddPatientDialog.razor](../../src/CTMS/CTMS/Components/Views/Commons/AddPatientDialog.razor) 與瀏覽篩選 [BrowseView.razor.cs:401-403](../../src/CTMS/CTMS/Components/Views/Commons/BrowseView.razor.cs) 都共用 `DropDownListDataService.Get院別()`，因此院別下拉只需改 `Get院別()` 一處即可同時生效。
- **AdminTools 流水號維護畫面** [LastSubjectNoView.razor](../../src/CTMS/CTMS/Components/Views/AdminTools/LastSubjectNoView.razor) 以三家硬編碼欄位顯示 / 寫回流水號（第 19-88 行）；若要在後台調整新院流水號，需為新院新增對應欄位與繫結。
- **首頁 / 頁面標題** [Home.razor:7](../../src/CTMS/CTMS/Components/Pages/Home.razor) 與 [BrowserPage.razor:17](../../src/CTMS/CTMS/Components/Pages/ClinicalInformation/BrowserPage.razor) 標題為「成大醫院」屬靜態文字，與多院統計無關，視需求才調整。

## 4. 操作步驟

建議依下列順序修改，先定義常數與資料、再改邏輯、最後改畫面，以降低編譯與資料不一致風險：

1. **常數**：在 [MagicObjectHelper.cs](../../src/CTMS/CTMS.Share/Helpers/MagicObjectHelper.cs) 新增 `prefix範例醫院`、`PrefixSheetName範例醫院`、`Sheet範例Early/Advance`（必要時抽血檔名常數）。
2. **資料檔**：
   - 在 [RandomList.xlsx](../../src/CTMS/CTMS/Data/RandomList.xlsx) 加入「範例Early」「範例Advance」分頁並填妥隨機碼。
   - 在 [SubjectNoGenerator.json](../../src/CTMS/CTMS/Data/SubjectNoGenerator.json) 加入 `"XXX範例":0`。
   - 注意：`RandomList` 改用 `RandomListRuntime.json` 作為執行期快取，若已存在須一併更新或刪除以重新讀取 xlsx（見 [RandomListService.cs:13-27](../../src/CTMS/CTMS.ExcelUtility/Services/RandomListService.cs)）。
3. **編號 / 隨機模型**：更新 [SubjectNoGeneratorModel.cs](../../src/CTMS/CTMS.DataModel/Models/ClinicalInformation/SubjectNoGeneratorModel.cs)、[SubjectNoGeneratorService.cs](../../src/CTMS/CTMS.Business/Services/ClinicalInformation/SubjectNoGeneratorService.cs)、[RandomParameterMode.cs](../../src/CTMS/CTMS.DataModel/Models/ClinicalInformation/RandomParameterMode.cs)、[RandomListService.cs](../../src/CTMS/CTMS.ExcelUtility/Services/RandomListService.cs)。
4. **院別對應 / 反查**：更新 [PatientService.cs](../../src/CTMS/CTMS/Services/PatientService.cs)、[SubjectNoHelper.cs](../../src/CTMS/CTMS.Business/Helpers/SubjectNoHelper.cs)、[PatientSingleRowCsvExportService.cs](../../src/CTMS/CTMS.Business/Services/ClinicalInformation/PatientSingleRowCsvExportService.cs)。
5. **儀表板**：更新 [DashboardService.cs](../../src/CTMS/CTMS.Business/Services/DashboardService.cs)（清單、容器、`GetHospitalName`），必要時補 [dashboard.js](../../src/CTMS/CTMS/wwwroot/js/dashboard.js) fallback。
6. **下拉與畫面**：更新 [DropDownListDataService.cs](../../src/CTMS/CTMS.Business/Services/ClinicalInformation/DropDownListDataService.cs) 的 `Get院別()`、[RandomPage.razor](../../src/CTMS/CTMS/Components/Pages/AdminTools/RandomPage.razor) 分頁，（如需後台維護）[LastSubjectNoView.razor](../../src/CTMS/CTMS/Components/Views/AdminTools/LastSubjectNoView.razor) 欄位。
7. 依專案規範更新 `appsettings.json` 版本號（Patch +1）並更新相關文件。

## 5. 驗證檢查清單

- [ ] 新增病人對話框院別下拉出現「範例醫院」。
- [ ] 選「範例醫院」收案後，受試者編號為 `XXX0001` 並逐次遞增；`SubjectNoGenerator.json` 的 `XXX範例` 計數同步增加。
- [ ] 隨機分組能從新院的 Early/Advance × EC/OC 分頁取得隨機碼並回填 Treatment（AI/Dr）。
- [ ] 抽血檢驗能依新院抓到正確的 `抽血檢驗血液N.json` / `抽血檢驗生化N.json`（`SubjectNoHelper.GetBloodFilename`）。
- [ ] 進入 `/Dashboard`：合作醫院卡片數變為新家數、副標含「範例」；醫院統計長條圖出現「範例」一欄且病例數正確（驗證 `GetHospitalName` 有歸戶到位）。
- [ ] 瀏覽頁院別篩選可勾選「範例醫院」並正確過濾。
- [ ] CSV 單列匯出時院名欄位正確顯示「範例醫院」。
- [ ] RandomPage 後台出現新院四個分頁。

## 6. 注意事項與風險

- **正規化漏接最常見**：`DashboardService.GetHospitalName` 與 `SubjectNoHelper.GetHospital` 是連續 `if`，新院若漏改，病人不會出錯但會被「靜默漏算」於醫院長條圖或抽血檔名比對，難以察覺。新增醫院務必同步這兩處。
- **編號連續性**：`SubjectNoGenerator.json` 為單一檔案、無交易控制；計數欄位鍵名須與 `SubjectNoGeneratorModel` 屬性序列化名完全一致，否則讀回時對不上而從 0 起算，造成編號重號。
- **隨機表配置**：`RandomList` 以 `RandomListRuntime.json` 作執行期快取，僅在快取不存在時才重讀 xlsx。新增分頁後若舊快取仍在，新院隨機碼不會生效，需刪除或重建快取。`ReadSheet` 預設讀到第 2000 列，分頁列數請勿超過。
- **既有資料**：既有三院的 `Patient.醫院` 值與 `SubjectNo` 前綴已落地，請勿更動既有常數字串（`NCKUH/CHIMEIH/KGH`、`成大/奇美/郭綜合`），只做「新增」不做「改名」，以免既有統計與反查全面失準。
- **目標收案人數**：儀表板完成率分母 `TargetCases` 固定為 294（見 [DashboardViewModel.cs:50](../../src/CTMS/CTMS.DataModel/Models/Dashboard/DashboardViewModel.cs)），新增醫院若改變總目標人數，須一併調整。
