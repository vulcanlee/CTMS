# 解決方案架構

解決方案檔：[src/CTMS/CTMS.sln](../../src/CTMS/CTMS.sln)，共 9 個專案。本文件說明各專案職責與相依關係。

## 1. 專案一覽

| 專案 | 類型 | 職責 |
| --- | --- | --- |
| `CTMS` | ASP.NET Core Web（Blazor Server） | 主網站：UI、路由、頁面組裝、DI 註冊、middleware、靜態檔、app 啟動 |
| `CTMS.Business` | Class Library | 商業邏輯核心：AI 整合、DICOM/影像、儀表板統計、角色權限、領域服務 |
| `CTMS.DataModel` | Class Library | DTO、ViewModel、前端交換模型、臨床資料 JSON 對應模型 |
| `CTMS.EntityModel` | Class Library | EF Core `BackendDBContext`、實體、Migration |
| `CTMS.ExcelUtility` | Class Library | Excel/CSV 讀寫：隨機表、風險評估輸入輸出、匯出大表 |
| `CTMS.Share` | Class Library | 共用 helper、常數、擴充方法（最底層、無相依） |
| `AIAgent` | Worker Service | 背景服務：輪詢佇列資料夾執行 AI 流程 |
| `AIAgent.Business` | Class Library | AIAgent 的商業邏輯與資料夾處理 |
| `EncodeJSON` | Console App | JSON 編碼/轉換小工具 |

## 2. 相依關係

以 `.csproj` 的 `ProjectReference` 為準：

```
CTMS (Web)
└── CTMS.Business
    ├── AIAgent.Business
    │   ├── CTMS.DataModel
    │   └── CTMS.ExcelUtility
    ├── CTMS.DataModel ──── CTMS.Share
    ├── CTMS.EntityModel ── CTMS.Share
    └── CTMS.ExcelUtility ─ CTMS.DataModel

AIAgent (Worker)
└── AIAgent.Business
    ├── CTMS.DataModel
    └── CTMS.ExcelUtility
```

重點觀察：

- **`CTMS.Share` 是最底層**，不相依任何專案；常數與 helper 放這裡。
- **`CTMS.Business` 是商業邏輯樞紐**，匯集所有資料/工具專案，並引用 AI/DICOM 套件（Azure.AI.OpenAI、fo-dicom）。
- **`AIAgent.Business` 同時被 Web 端（透過 CTMS.Business）與背景服務（AIAgent）引用**，是主站與背景服務共用 AI 處理邏輯的橋樑。
- **`EncodeJSON`** 為獨立工具，不在主執行路徑上。

## 3. 主網站 UI 組織

`CTMS` 專案內的畫面分層（詳見各功能模組文件）：

- `Components/Pages` — 對應路由的頁面（如 `BasicClinical/{code}`）。
- `Components/Views` — 頁面內主要功能 View（如 `SurgeryView`、`BloodTestView`）。
- `Components/Commons` — 共用對話框、提示、picker、上傳元件。
- `Components/Layout` — Layout、Header/Footer、導覽骨架（含 `BottomNavigationView` 串接個案工作頁）。

## 4. 應用程式啟動行為（CTMS）

主站啟動（`Program.cs`）時會：

1. 設定 NLog。
2. 註冊 Razor Components / MudBlazor / Syncfusion / AntDesign 與各領域服務（DI）。
3. 設定 SQLite `BackendDB.db` 並執行 `dbContext.Database.Migrate()`。
4. 建立預設專案 `Default Project`、預設角色 `Default Role`、預設新帳號角色、預設管理者帳號 `support`。
5. 將沒有狀態的病患預設為「收案」。

> 安全提醒：預設管理者帳密為 `support` / `support`，僅適合開發。正式環境須立即變更。

## 5. 重要執行期目錄

主站把下列資料夾作為靜態內容或工作目錄（見 `CTMS.csproj` 與 `Program.cs`）：

| 目錄 | 用途 |
| --- | --- |
| `UploadFiles` | 上傳影像/檔案 |
| `UploadTemp` / `DecompressFiles` | 上傳暫存、解壓縮 |
| `DawnCache` | DICOM 影像快取 |
| `DownloadFiles` | 下載輸出 |
| `PdfFiles` | PDF 輸出 |
| `OperationContent` | 操作歷程內容 |
| `Data` | 問卷、檢驗、隨機表等資料檔 |

AI 推論相關的佇列與推論路徑由 `AgentSetting` 設定（`C:\temp\...`、`C:\CanWellBeing\...`），詳見 [06-部署與維護/部署與環境設定.md](../06-部署與維護/部署與環境設定.md)。
